How it works

Four steps, one honest trick

No training, no fine-tuning, no black box. Behaviour is scored with plain arithmetic; the LLM only ranks, assembles, and explains ├бРѓгРђЮ it's never allowed to invent.

1

Score

Order history ├бРђаРђЎ three behaviour dials. The dominant dial picks the strategy.

2

Retrieve

RAG filters 8,680 real restaurants (86,800 dishes, 9 cities) to ~15 grounded candidates.

3

Generate

Gemini assembles the homepage as strict JSON ├бРѓгРђЮ a reason on every item.

4

Serve

Facts re-verified against the database; cached 30 min per user per meal slot.

repeat  = orders repeating a (restaurant, dish) pair ├Ѓ┬и total  // the usual
variety = distinct dishes ├Ѓ┬и visits, where visits ├бРђ░┬Ц 2        // loyal but curious
explore = distinct restaurants ├Ѓ┬и total orders            // never the same place
persona = argmax(normalise(repeat, variety, explore))

Decisions I'd defend

The judgment calls behind it

Why LLM + RAG

Rules can't reason; an ungrounded LLM invents restaurants. Retrieval feeds it only real, orderable places ├бРѓгРђЮ grounded and smart.

Degrade, never break

Model waterfall with retries; if all models are busy, a handsome non-AI homepage ships instead of an error.

Facts stay verified

The AI writes the story; the database overwrites the numbers ├бРѓгРђЮ ratings and delivery times are never the model's word.

Diet is sacred

A declared diet is an unbreakable constraint. Jain and vegan are never guessed ├бРѓгРђЮ guessing wrong is real harm.

Location is consent

Listings need live GPS permission ├бРѓгРђЮ no quiet fallback to a stored address you may not be at.

Cost by design

30-min homepage cache per user per slot; 24-hour trend cache. AI cost scales with people, not page loads.
